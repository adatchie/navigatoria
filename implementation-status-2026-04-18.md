# 実装状況まとめ 2026-04-18

## 概要

現状の `navigatoria` は、航海・寄港・交易・港サービス・遭遇・仮戦闘までを一通り遊べる段階に入っている。  
一方で、洋上地図と港座標、出航時の沿岸挙動、狭い海峡の通行性など、地理系の土台はまだ調整中であり、今後の実装はこの基盤安定化を優先する必要がある。

---

## 現在の実装状況

### 1. ゲーム基盤

- React + Vite + Three.js + React Three Fiber で構成
- Zustand でゲーム状態、航海状態、ワールド状態、経済状態、クエスト状態を管理
- Dexie ベースの保存/読込あり
- GitHub Actions で `lint` / `build` を自動実行
- GitHub Pages 上でプレイ可能

### 2. 航海

- 風向・風速を使った帆走モデルあり
- 帆の開閉率による速力変化あり
- 舵クリックで方位変更可能
- 港の出航 / 入港あり
- 航海中の食料・水消費あり
- 食料/水不足による士気低下・船員減少・耐久低下あり
- 乗組員不足・士気低下・積載率による速度/旋回補正あり
- 航海イベントあり
  - 雨水回収
  - 釣果
  - 漂流物回収
  - 船内トラブル
  - 索具損傷
  - 体調不良
  - スコール

### 3. 地図・地形

- Natural Earth ベースの地形描画を導入済み
- 港座標は `lon/lat -> Mercator -> world` で再投影
- 港位置は海側へ最低限補正する仕組みあり
- 世界の左右反転は表示系で修正済み
- ただし以下は未安定
  - 港位置の個別微調整
  - 出航直後に沿岸へ引っかかるケース
  - 狭い海峡での通行性
  - 地図縮尺の最終値

### 4. 港

- 港画面あり
- メニューは以下に整理済み
  - 港概要
  - 出航所
  - 市場
  - ギルド
  - 酒場
  - 造船所
  - 銀行
  - 保管庫
- 港ごとの施設有無で表示項目を切り替え

### 5. 経済・交易

- 港ごとの市場在庫
- 購入/売却
- 在庫や影響度を加味した価格変動
- 投資システム
- 銀行
  - 預金
  - 引き出し
  - 借入
  - 返済
  - 利子処理

### 6. クエスト

- 交易クエスト実装済み
  - 輸送
  - 買い出し
  - 売り込み
- 期限・ランク・報酬あり
- 報酬
  - 金
  - 経験値
  - 名声
  - 影響度
  - アイテム
- 進行/納品/報告まで一連の流れあり

### 7. 酒場・補給・修理

- 酒場で士気回復
- 船員雇用
- 食料/水補給
- 応急修理 / 通常修理 / オーバーホール

### 8. 造船所

- 旗艦切替
- 艤装強化

  - 船倉補強

### 9. 遭遇・仮戦闘

## 直近で入れた修正

### 地図・操舵・見え方

- 表示上の左右反転を修正
- 港マーカーの見え方を調整
- 初期カメラ向きを調整
- 船モデルの向きを地図表示に合わせて修正

### 地形通行

- 狭い海岸沿いで引っかかった際、陸地に当たったらその場で即停止するだけでなく、接線方向へ滑らせる処理を追加
- 出航時は海側へ少し押し出し、最初の数歩が海である出航点を探す処理を追加

### 遭難・航行不能

- 船員全滅時はそのまま海上停止ではなく、出航元の港へ強制送還する処理を追加
- 遭遇後に船員や耐久が減った時に通知を出すよう修正
- 船員不足で進めない時に、その理由が HUD と通知で分かるように修正

## 現在残っている主な問題

### 1. 地理基盤

- 一部港座標がなお海岸線と噛み切れていない
- リスボン / セビリア / ボルドー / ロンドンのような沿岸・河口港は個別港湾座標の調整余地がある
- ジブラルタル海峡の通行性がまだ不安定
- 地中海内でも沿岸判定で進めなくなるケースがある

### 2. 航海UX

- 停止理由が見えやすくはなったが、沿岸滑り処理はまだ簡易実装
- 海峡や湾内での経路選択は本来、将来的には「衝突回避つきの連続操船」に寄せるべき

### 3. 戦闘

- 現状は仮戦闘ループ
- 実際の位置取り、射角、接舷、白兵画面は未実装

### 4. 地図と実港位置

- 現在は実地図投影ベースだが、港座標は「実港湾座標」へさらに寄せる余地がある
- 本当に完成度を上げるなら、主要港は city center ではなく harbor mouth / roadstead 基準で見直したほうがよい

## 補足

現状は「遊べる機能」はかなり揃っているが、  
品質上の最大ボトルネックは **地図・港・沿岸挙動の整合性** である。

したがって、次の数ターンは新機能の追加よりも、

- 地図基盤の安定化
- 航行不能の見える化
- 海峡・港周辺の通行性改善

を優先するのが最も効果的。
